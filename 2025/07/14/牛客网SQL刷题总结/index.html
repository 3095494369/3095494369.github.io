总结-基础知识

牛客网在线编程_SQL篇_SQL快速入门

去重复数据

关键字：distinct

select distinct university 
from user_profile

过滤空值

where 列名 is not 'null' 
where 列名 != 'null'
where 列名 <> 'null'

多个匹配值

关键字：in

select device_id,gender,age,university,gpa
from user_profile
where university in ('北京大学','复旦大学','山东大学')

模糊查询（匹配字符串）

关键字：like 或者 not like

select device_id,age,university
from user_profile
where university like '北京%'

北京%：列名以北京开头

%北京：列名以北京结束

%北京%：列名中包括北京

MAX()最大函数值

关键字：max()

select max(gpa) as gpa
from user_profile
where university = '复旦大学'

avg()求平均值

select  count(gender) as male_num,round(avg(gpa),1) as avg_gpa
from user_profile
where gender = 'male'

round()四舍五入

round(avg(gpa),1)其中第二个参数表示保留一位小数

select  count(gender) as male_num,round(avg(gpa),1) as avg_gpa
from user_profile
where gender = 'male'

排序

关键字：order by 字段 ASC

​ ASC:默认 升序

​ DESC: 降序

——对多个字段排序时，按照书写的先后顺序依次排列

select
    gender,university,count(device_id) as user_num,round(avg(active_days_within_30),1) as avg_active_day,round(avg(question_cnt),1) as avg_question_cnt
from
    user_profile
group by gender,university
order by gender ASC,university ASC

先按照gender(性别)排序，在每个性别里面再按照university字段排序

分组

关键字：group by 字段

多个字段使用,分隔

select
    gender,university,count(device_id) as user_num,round(avg(active_days_within_30),1) as avg_active_day,round(avg(question_cnt),1) as avg_question_cnt
from
    user_profile
group by gender,university
order by gender ASC,university ASC

对分组的字段，条件查询

关键字：having

当对字段进行group by分组之后，就不能使用where来限制条件，必须使用having

select
    university,
    round(avg(question_cnt), 3) as avg_question_cnt,
    round(avg(answer_cnt), 3) as avg_answer_cnt
from
    user_profile
group by
    university
having 
    avg(question_cnt)<5 or avg(answer_cnt) <20

多表连接 21-25

子查询

注意点：子查询中不存在多表之间的联系，是使用in等关键字匹配字段的值来连接。

即 from关键字后都只有一个表

select device_id,question_id,result
from
    question_practice_detail
where 
question_practice_detail.device_id in
(
    select device_id
    from user_profile
    where university = '浙江大学'
)

内连接–INNER JOIN

关键字：INNER JOIN 连接的表名 on 关联的字段

后续可使用where关键字作筛选

select question_practice_detail.device_id,question_id,result
from
    question_practice_detail
Inner JOIN user_profile 
on question_practice_detail.device_id = user_profile.device_id
where university = '浙江大学'

在使用多个INNER JOIN关键字内连接其他表时，第一个表的字段与第二个表的字段连接，第二个表的字段与第三个表的字段连接。

左右连接29

当结果中出现某一列需要输出为null的字段值，这么就使用左右连接

select Products.prod_name,OrderItems.order_num
from Products
left JOIN OrderItems
on Products.prod_id = OrderItems.prod_id 
order by prod_name

SELECT
    ROUND(
        SUM(CASE WHEN p2.device_id IS NOT NULL THEN 1
                ELSE 0
            END
        ) / COUNT(*),
        4
    ) AS avg_ret
    # 也可以表示为
    # COUNT(q2.device_id) / COUNT(q1.device_id) AS avg_ret
    # 对于count（）函数计数时不会统计值为null的条目
    # 因为是最左连接，会将p1表中的数据都显示，而只会显示符合条件的p2中留存的数据，否则为空
        # 也就是说对于同一个人来说，第一天刷题，才会有p1.device_id。第二天刷题了，才会有p2.device_id，否则为空
FROM
    (
        SELECT 
            DISTINCT device_id,
            date
        FROM
            question_practice_detail
    ) p1
LEFT JOIN (
        SELECT 
            DISTINCT device_id,
            date
        FROM
            question_practice_detail
    ) p2 
ON p1.device_id = p2.device_id AND p2.date = DATE_ADD(p1.date, INTERVAL 1 DAY);

内连接与左右连接的区别

连接类型 行为 适用场景
INNER JOIN 只返回两个表中匹配的行 只关心两个表都存在的关联数据
LEFT JOIN 返回左表所有行 + 右表匹配的行（右表无匹配则补 NULL） 即使右表没有匹配，也要保留左表数据
RIGHT JOIN 返回右表所有行 + 左表匹配的行（左表无匹配则补 NULL） 即使左表没有匹配，也要保留右表数据

联合查询 25

关键字：UNION ALL 或UNION

UNION:表示将两个查询结果连接起来，并且去重

UNION ALL:表示将两个查询结果连接起来，但是不去重

select device_id,gender,age,gpa
from user_profile
where university = '山东大学'
union ALL
select device_id,gender,age,gpa
from user_profile
where gender = 'male
  • 结果集：包含两类数据的并集：
    1. 山东大学的所有学生（无论性别）。
    2. 所有男性学生（无论学校）。
  • 重复数据：如果一个山东大学的学生同时也是男性，则该学生的记录会 出现两次（因为 UNION ALL 保留重复）。

条件逻辑

关键字：

CASE
 WHEN condition1 THEN result1
 WHEN condition2 THEN result2
 ...
 ELSE default_result
END

当符合条件condition1时，将他设置为result1

select
    case
        when
            age < 25
            or age is null then '25岁以下'
            else '25岁及以上'
    end as age_cut,
    count(device_id) as number

窗口函数33,41

窗口函数

SELECT *, DENSE_RANK() OVER (PARTITION BY university ORDER BY gpa) AS dr
FROM user_profile

# 严格连续编号，即使分数相同也分配不同序号。ROW_NUMBER()

# 每个大学中GPA最低的所有学生（包括并列情况）：DENSE_RANK()
SELECT device_id, university, gpa
FROM (
    SELECT *, DENSE_RANK() OVER (PARTITION BY university ORDER BY gpa) AS dr
    FROM user_profile
) AS univ_min
WHERE dr = 1
ORDER BY university

一、聚合类窗口函数

SUM(), AVG(), MIN(), MAX(), COUNT()

二、排名类窗口函数

ROW_NUMBER() 行号（相同值不同号） 1,2,3,4
RANK() 排名（相同值并列但留空） 1,2,2,4
DENSE_RANK() 密集排名（相同值并列无空） 1,2,2,3

三、偏移分析类窗口函数

LAG(column, n) -- 访问前n行值
LEAD(column, n) -- 访问后n行值
FIRST_VALUE(column) -- 窗口第一行
LAST_VALUE(column) -- 窗口最后一行

四、统计类窗口函数

PERCENT_RANK() -- 排名百分比 (0-1)
CUME_DIST()   -- 累计分布 (0-1)
STDDEV(), VAR() -- 标准差和方差

关键字

PARTITION BY —— 分组处理

  • 类似 GROUP BY，但不聚合，仅把行划分成若干组；
  • 每组窗口函数单独计算；
  • 可选，不写则默认全表一组。

ORDER BY —— 窗口内排序

  • 决定行的先后顺序；
  • 排序顺序直接影响如排名、前后行等函数计算；
  • 必须写在 OVER(...) 内部，否则是整个结果集排序。

ROWS BETWEEN —— 明确窗口范围

ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW 从分区第一行到当前行（累计）
ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING 当前行、前一行、后一行
ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING 当前行到最后一行

若不写 ROWS，窗口函数默认是“从分区首行到当前行”或整个分区（函数而异）

if条件表达34

IF(condition, value_if_true, value_if_false)
这是一个条件表达式，类似于编程语言中的三元运算符：

sum(if(qpd.result='right', 1, 0))
  • qpd.result='right'：条件判断，检查 result 字段是否等于 ‘right’
  • 1：如果条件为真（即 result=’right’）时返回的值
  • 0：如果条件为假（即 result≠’right’）时返回的值
select up.device_id,university,
    count(question_id) as question_cnt,
    sum(if(qpd.result='right', 1, 0)) as right_question_cnt
from user_profile as up

left join question_practice_detail as qpd
on qpd.device_id = up.device_id and month(qpd.date) = 8

where up.university = '复旦大学'
group by up.device_id

正则表达式40

关键字：regexp

  1. 电话号码必须是 10 位数字。
  2. 电话号码的第一位不能以 0 开头。
  3. 电话号码的格式可以是连续的 10 位数字，或以-分隔的格式（如123-456-7890）。
select *
from contacts
where phone_number regexp '^[1-9][0-9]{9}$|^[1-9][0-9]{2}-[0-9]{3}-[0-9]{4}$'
order by id ASC

这个正则表达式的作用如下：

  • ^ 表示匹配字符串的开头，$ 表示匹配字符串的结尾，确保整个字符串都符合指定的模式。
  • [1-9] 表示电话号码的第一位不能是 0，必须是 1 到 9 之间的数字。
  • [0-9]{9} 表示接下来的 9 位可以是任意数字，与开头的 [1-9] 组合形成一个完整的 10 位数字电话号码。
  • | 是正则表达式的“或”操作符，用于分隔两种不同的匹配模式。
  • ^[1-9]{2}-[0-9]{3}-[0-9]{4}$ 表示电话号码可以是以 “-” 分隔的格式，其中第一部分是 3 位数字，第二部分是 3 位数字，第三部分是 4 位数字。

Limit关键字

分页查询

关键字：LIMIT pageSize OFFSET (pageNumber - 1) * pageSize

SELECT * FROM Customers
LIMIT 10 OFFSET 20;

表示1：

  • 跳过前 20 行（OFFSET 20）
  • 返回接下来的 10 行

等价于取出第 21～30 行。

表示2：

  • 第 3页，每页 10 条数据

取前 N 条最大/最小的值

关键字：LIMIT + ORDER BY

-- 取销量最高的 3 个商品
SELECT product_name, sales
FROM Products
ORDER BY sales DESC
LIMIT 3;

技巧注意

  1. 是或不是的时候才能用**“is”，别的时候有关联关系还是用等号和大于小于**才行

  2. select关键字后面字段之间使用,间隔，但是最后一个字段后面不能加,

  3. 如果from字段后面有多个表，那么在where语句中就必须写出这多表之间的联系。

  4. 如果是多表连接，最后的查询结果必须指明是那个表里面的字段（如果这两个表都有该字段）。

  5. 在使用多个INNER JOIN关键字内连接其他表时，第一个表的字段与第二个表的字段连接，第二个表的字段与第三个表的字段连接。

    select  user_profile.university,question_detail.difficult_level,ROUND(COUNT(question_practice_detail.question_id) / COUNT(DISTINCT user_profile.device_id), 4) AS avg_answer_cnt
FROM user_profile
INNER JOIN question_practice_detail
on user_profile.device_id = question_practice_detail.device_id
INNER JOIN question_detail
on question_practice_detail.question_id = question_detail.question_id
group by university,difficult_level

  6. 多表连接中select，group by , order by 子句中的字段最好加上表名
